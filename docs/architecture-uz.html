🎯 Muammo bayoni

MCP DevBridge hal qilishga yaratilgan muammolar

Zamonaviy AI dasturlash ish oqimlarida tarqoqlik muammosi mavjud: har bir vosita (GitHub, ma'lumotlar bazalari, fayl tizimlari) o'ziga xos API, autentifikatsiya mexanizmi va ma'lumotlar formatiga ega. Dasturchilar xususiyatlar yaratish o'rniga "yopishqoq" kod yozishga vaqt sarflaydi.

  • AI vositalar uchun dasturchi resurslari bilan ishlashning yagona protokoli yo'q — har bir integratsiya maxsus kod talab qiladi.
  • Real vaqt kuzatuvining yo'qligi: dasturchilar bir joyda bir nechta integratsiya bo'yicha vosita chaqiruvlarini, kechikishni yoki xatolarni kuzata olmaydi.
  • Markazlashgan audit jurnali bo'lmagan holda AI yordamchilar sezgir repozitoriy yoki ma'lumotlar bazalariga kirishidagi xavfsizlik bo'shliqlari.
  • Mahalliy ishlab chiqish qiyinligi: GitHub, ma'lumotlar bazalari va fayl tizimlari uchun alohida mock serverlarni ishga tushirish vaqt talab qiladi.
  • AI modellariga yangi vositalarni ochishning standartlashtirilgan usuli yo'q — har bir model sotuvchisi o'zining funksiya chaqiruv formatiga ega.

AI vositalarini dasturchi muhitingizga integratsiya qilish uchun zarur bo'lgan hamma narsa

🚀 Tezkor boshlash

MCP DevBridgeni 5 daqiqadan kam vaqtda mahalliy ishga tushirish

1-qadam: Repozitoriyni klonlash

git clone https://github.com/your-org/mcp-devbridge.git cd mcp-devbridge

2-qadam: Muhit o'zgaruvchilarini sozlash

cp .env.example .env # .env faylini tahrirlang va GitHub tokenini kiriting: GITHUB_TOKEN=ghp_your_personal_access_token DB_CONNECTION=Data Source=devbridge.db

3-qadam: Docker Compose bilan barcha xizmatlarni ishga tushirish

docker compose up --build -d # Xizmatlar ishga tushmoqda: # ✓ dashboard → http://localhost:3000 # ✓ api-gateway → http://localhost:5000 # ✓ mcp-github → http://localhost:3001 # ✓ mcp-database → http://localhost:3002 # ✓ mcp-filesystem → http://localhost:3003

4-qadam: Vosita chaqiruvini sinab ko'rish

curl -X POST http://localhost:5000/api/mcp/call \ -H "Content-Type: application/json" \ -d '{"server":"github","tool":"list_repos","params":{}}'

5-qadam: Dashboard-ni http://localhost:3000 da oching

🔒 Xavfsizlik mulohazalari

Ishlab chiqarish joylashuvi uchun o'rnatilgan himoya vositalari

🔑 Token boshqaruvi

GitHub tokenlari faqat muhit o'zgaruvchilarida saqlanadi, hech qachon manba kodida yoki ma'lumotlar bazasida emas. Tokenlar API javoblarida qaytarilmaydi.

📋 Audit jurnali

Har bir vosita chaqiruvi chaqiruvchi identifikatori, parametrlar, javob vaqti va HTTP holati bilan saqlanadi. Jurnallar yozilgandan keyin o'zgartirib bo'lmaydi.

🗂 FileSystem qum qutisi

FileSystem MCP serveri sozlanishi mumkin bo'lgan ildiz katalogida ishlaydi. Yo'l kesib o'tish urinishlari (../) tekshiruv qatlamida bloklanadi.

🛡 So'rov xavfsizligi

Ma'lumotlar bazasi vositasi o'qish so'rovlarini yozish so'rovlaridan ajratadi. DDL iboralari (DROP, CREATE) query_table vositasida sukut bo'yicha bloklanadi.

🏥 Holat monitoringi

Fon holati tekshiruvlari server nosozliklarini 30 soniya ichida aniqlaydi. Nosog'lom serverlar tiklanguncha marshrutlashdan chiqarib tashlanadi.

🐳 Tarmoq izolyatsiyasi

Docker Compose barcha xizmatlarni ichki tarmoqqa joylashtiradi. Faqat dashboard (3000) va gateway (5000) xostga ochiq.
