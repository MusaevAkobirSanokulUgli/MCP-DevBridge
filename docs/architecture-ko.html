🎯 문제 정의

MCP DevBridge가 해결하기 위해 만들어진 문제들

현대 AI 개발 워크플로우는 파편화 문제를 겪고 있습니다. GitHub, 데이터베이스, 파일 시스템 등 각 도구가 고유한 API, 인증 메커니즘, 데이터 형식을 가지고 있어 개발자들이 기능 개발 대신 연결 코드를 작성하는 데 시간을 낭비합니다.

  • AI 도구가 개발자 리소스와 상호작용하기 위한 통합 프로토콜이 없어 모든 통합이 맞춤 코드를 필요로 합니다.
  • 실시간 관찰 가능성 부재: 개발자가 여러 통합에 걸친 도구 호출, 지연 시간, 오류를 한 곳에서 모니터링할 수 없습니다.
  • 중앙화된 감사 로깅 없이 AI 어시스턴트가 민감한 저장소나 데이터베이스에 접근할 때의 보안 취약점.
  • 로컬 개발의 어려움: GitHub, 데이터베이스, 파일 시스템을 위한 개별 목업 서버를 구동하는 것은 시간이 많이 소요됩니다.
  • AI 모델에 새 도구를 노출하는 표준화된 방법이 없어 각 모델 벤더마다 다른 함수 호출 형식을 사용합니다.

✨ 주요 기능

AI 도구를 개발자 환경에 통합하는 데 필요한 모든 것

📡
실시간 대시보드
SignalR WebSocket을 통한 라이브 도구 호출 모니터링. 세 개의 MCP 서버 전체에서 호출, 지연 시간, 오류를 단일 통합 뷰로 확인합니다.
🐙
GitHub MCP Server
도구: list_repos, get_repo, create_issue, list_issues, get_pull_requests.
🗄
Database MCP Server
도구: list_tables, describe_schema, query_table, execute_query. 안전한 읽기/쓰기 분리.
📁
FileSystem MCP Server
도구: list_files, read_file, write_file, search_files. 샌드박스 디렉터리 접근.
📋
API Gateway & 감사 로깅
모든 도구 호출이 타임스탬프, 메서드, 파라미터, 응답 시간, 상태와 함께 기록됩니다. EF Core를 통해 SQLite에 전체 감사 추적이 저장됩니다.
💓
백그라운드 헬스 체크
호스티드 서비스가 30초마다 세 MCP 서버를 핑합니다. 상태 변화는 SignalR을 통해 연결된 대시보드 클라이언트로 브로드캐스트됩니다.

🚀 빠른 시작

5분 이내에 MCP DevBridge를 로컬에서 실행하기

1단계: 저장소 복제

git clone https://github.com/your-org/mcp-devbridge.git cd mcp-devbridge

2단계: 환경 변수 설정

cp .env.example .env # .env를 편집하여 GitHub 토큰을 설정하세요: GITHUB_TOKEN=ghp_your_personal_access_token DB_CONNECTION=Data Source=devbridge.db

3단계: Docker Compose로 모든 서비스 시작

docker compose up --build -d # 서비스 시작 중: # ✓ dashboard → http://localhost:3000 # ✓ api-gateway → http://localhost:5000 # ✓ mcp-github → http://localhost:3001 # ✓ mcp-database → http://localhost:3002 # ✓ mcp-filesystem → http://localhost:3003

4단계: 도구 호출 테스트

curl -X POST http://localhost:5000/api/mcp/call \ -H "Content-Type: application/json" \ -d '{"server":"github","tool":"list_repos","params":{}}'

5단계: http://localhost:3000에서 대시보드 열기

🔒 보안 고려사항

프로덕션 배포를 위한 내장 보호 장치

🔑 토큰 관리

GitHub 토큰은 환경 변수에만 저장되며, 소스 코드나 데이터베이스에는 절대 저장되지 않습니다. API 응답에서 토큰이 반환되지 않습니다.

📋 감사 로깅

모든 도구 호출이 호출자 ID, 파라미터, 응답 시간, HTTP 상태와 함께 저장됩니다. 로그는 한번 작성되면 변경할 수 없습니다.

🗂 파일 시스템 샌드박싱

FileSystem MCP 서버는 설정 가능한 루트 디렉터리 내에서만 작동합니다. 경로 순회 시도(../)는 유효성 검사 계층에서 차단됩니다.

🛡 쿼리 안전성

데이터베이스 도구는 읽기 쿼리와 쓰기 쿼리를 분리합니다. DDL 문(DROP, CREATE)은 기본적으로 query_table 도구에서 차단됩니다.

🏥 헬스 모니터링

백그라운드 헬스 체크가 30초 내에 서버 장애를 감지합니다. 비정상 서버는 회복될 때까지 라우팅에서 제외됩니다.

🐳 네트워크 격리

Docker Compose는 모든 서비스를 내부 네트워크에 배치합니다. 대시보드(3000)와 게이트웨이(5000)만 호스트에 노출됩니다.
